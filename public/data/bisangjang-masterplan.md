## 점수 시스템 개편 기획 (2026-04-11 기준)

### 확정 결정사항

#### 결정 #1: DART 데이터 없음 vs 실제 매출 없음 구분
- DART 미등록/수집실패 → 해당 항목 null 처리 → 나머지로 100점 환산
- DART 등록됐지만 실제 매출 0 → 0점 처리
- 적용 범위: 전 섹터 공통

#### 결정 #2: 매출성장률 계산 방식
- 3개년 데이터 → 3개년 CAGR
- 2개년 데이터 → 전년대비 성장률
- 1개년 데이터 → 해당 연도 매출 절대값으로 점수 계산
- 데이터 없음 → null (결정#1 적용)
- 적용 범위: 전 섹터 공통

#### 결정 #3: 라운드별 투자금액 점수 적용 방식
- 보유 라운드 중 가장 높은 라운드 1개 점수만 반영
- 하위 라운드 점수 모두 배제 (중복 합산 없음)
- 투자금액 상승(5점)은 별도 유지 — 전라운드 대비 독립 계산
- 적용 범위: 전 섹터 공통

#### 결정 #4~7: 커머스 섹터 고정값 항목 처리
- 시장크기: Claude AI 섹터분석 기반 동적 계산
- 경쟁구도: Claude AI marketCompetition 기반 동적 계산
- 팀·운영: 창업팀퀄리티 + 창업경과시간 통합 → 설립일 기준 동적 계산
- 대외신뢰도: 계약신뢰도 + 미디어노출 + 대외수상 통합 → 뉴스수집 기반

#### 결정 #8: 전 섹터 공통 데이터 우선순위 및 점수 처리
- 1순위: VC 투자 데이터
- 2순위: DART 기반 재무 데이터
- VC+DART 없음 → 점수 미정 ("데이터부족" 표기)
- VC만 있음 → VC 점수 비례로 채워서 100점 환산
- VC+DART 있음 → VC+DART 점수 비례로 채워서 100점 환산
- DART만 있음 → DART 점수 비례로 채워서 100점 환산
- 구현: normalizeScore() 공통 함수로 전 섹터 적용
