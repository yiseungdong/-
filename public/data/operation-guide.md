## 4. 백업 방법

### 자동 백업 (Render.com PostgreSQL)
- Render.com 유료 플랜: 자동 일일 백업 제공
- 무료 플랜: 수동 백업 필요

### 수동 백업 (무료 플랜용)
터미널에서 아래 명령어 실행:

```
pg_dump DATABASE_URL > backup_날짜.sql
```

예시:
```
pg_dump $DATABASE_URL > backup_20260401.sql
```

### 백업 복원
```
psql DATABASE_URL < backup_날짜.sql
```

### 백업 주기 권장
- 최소 주 1회 수동 백업
- 시즌 종료 전 반드시 백업
- 대규모 업데이트 전 반드시 백업

## 5. 장애 대응

### 서버 다운 시
1. Render.com 대시보드 확인 → 서버 상태
2. 로그 확인 → 에러 메시지 파악
3. 최근 배포가 원인이면 → 이전 버전으로 롤백
   - Render.com → Manual Deploy → 이전 커밋 선택
4. DB 문제면 → PostgreSQL 연결 문자열 확인
5. 해결 안 되면 → Render.com 서버 재시작 (Manual Restart)

### DB 연결 끊김
1. Render.com PostgreSQL 대시보드 확인
2. 연결 문자열(DATABASE_URL) 변경되었는지 확인
3. 환경변수 업데이트 필요 시 Render.com → Environment 탭에서 수정
4. 서버 재시작

### API 응답 느림
1. Render.com 로그에서 느린 쿼리 확인
2. 무료 플랜은 15분 비활동 시 슬립 → 첫 요청 느림 (정상)
3. 지속적이면 DB 인덱스 확인

### 긴급 연락
- Render.com 상태 페이지: https://status.render.com
- Render.com 지원: https://render.com/support

## 6. 업데이트 배포 절차
1. 로컬에서 코드 수정 + 테스트
2. `git add .`
3. `git commit -m "설명"`
4. `git push`
5. Render.com이 자동 배포 (2~3분 소요)
6. asteria.me.kr에서 확인
7. 문제 발생 시 롤백 (이전 커밋으로)

## 7. 주요 환경변수 목록
| 변수명 | 설명 | 예시 |
|--------|------|------|
| DATABASE_URL | PostgreSQL 연결 | postgres://user:pass@host/db |
| JWT_SECRET | JWT 비밀키 | 랜덤 문자열 32자 이상 |
| JWT_REFRESH_SECRET | 리프레시 토큰 비밀키 | 랜덤 문자열 32자 이상 |
| ANTHROPIC_API_KEY | Claude API 키 | sk-ant-... |
| PORT | 서버 포트 | 3000 (Render가 자동 설정) |
| NODE_ENV | 환경 | production |
